# 🔐 Rugano Prepaid Door Lock System

A modern, web-based solution for managing prepaid, rent-based door access control in Kenyan rental properties. The system automates tenant access based on M-Pesa rent payments and provides a secure, mobile-first admin dashboard for landlords and property managers.

## Overview

Rugano eliminates manual rent collection and key handover. Tenants pay via M-Pesa and the system automatically calculates how many days of access their payment grants. When access expires, the door locks. When they pay again, it unlocks — no landlord involvement required.

**Access Calculation:**

```
daily_cost     = monthly_rent ÷ 30
days_granted   = amount_paid ÷ daily_cost
access_expires = current_expiry + days_granted
```

**Example:** Unit rent KES 6,000/month → daily cost KES 200. Tenant pays KES 2,000 → gets 10 days access. Partial payments always accepted and extend from the current expiry date.

---

## How It Works

```
Tenant pays rent via M-Pesa
        ↓
Payment webhook hits /api/pay/confirm
        ↓
System calculates days granted and updates access_expires_at
        ↓
Tenant enters PIN on door keypad
        ↓
System checks PIN + access expiry via /door/check
        ↓
Door granted or denied
```

## Features

### Landlord / Caretaker Dashboard

- Secure login with JWT authentication
- Add and manage rental units with monthly rent per unit
- Add, deactivate, and manage tenants
- Change tenant PINs from the dashboard
- Request M-Pesa payment — STK Push directly to tenant's phone
- View full payment history with M-Pesa references
- View access logs — granted, denied, manual unlocks
- Manual unlock with mandatory authorization logging
- Real-time unit access status on the dashboard home

### Tenant Access

- PIN-based door entry (6–8 digit PIN)
- Access automatically granted on payment
- Access automatically revoked on expiry
- Exit always works regardless of payment status

### Payment System

- M-Pesa STK Push via Citapay API
- Webhook-based automatic access update on payment
- Idempotency check — duplicate callbacks are logged and ignored
- Validation endpoint for Safaricom C2B
- Internal payment simulator for development and testing

### Door Simulator

- Web-based keypad for testing PIN access
- Shows granted/denied status with tenant name and days remaining
- Accessible directly from the admin dashboard

### Mobile-First Design

- Bottom navigation bar on mobile devices
- Fully responsive across all screen sizes
- Works as a dashboard on the landlord's phone

### Door Access

| Method | Endpoint                     | Description                 |
| ------ | ---------------------------- | --------------------------- |
| POST   | `/door/check`                | Verify PIN and check access |
| GET    | `/door/status/{unit_number}` | Get unit access status      |
| GET    | `/door/simulator`            | Web-based door simulator    |

**POST /door/check — Request:**

```json
{
  "unit_number": "A1",
  "pin": "123456"
}
```

**Response — Granted:**

```json
{
  "access": "granted",
  "tenant_name": "Jane Wanjiku",
  "unit": "A1",
  "expires": "24 Mar 2026, 01:19",
  "days_remaining": 8,
  "reason": "Welcome Jane Wanjiku. Access valid until 24 Mar 2026."
}
```

**Response — Denied:**

```json
{
  "access": "denied",
  "reason": "Access expired on 14 Mar 2026. Please pay rent."
}
```

## Environment Variables

```env

# JWT
```

| Variable                      | Description                                |
| ----------------------------- | ------------------------------------------ |
| `SUPABASE_URL`                | Supabase project URL                       |

| `SUPABASE_SECRET_KEY`         | Supabase secret key (backend only)         |
| `SECRET_KEY`                  | JWT signing secret                         |
| `ALGORITHM`                   | JWT algorithm (default: HS256)             |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | Session duration in minutes                |
| `CITAPAY_SECRET_KEY`          | Citapay API key (`sk_test_` or `sk_live_`) |
| `CITAPAY_WEBHOOK_SECRET`      | Citapay webhook signing secret             |

| `ENVIRONMENT`                 | `development` or `production`              |

## Hardware Integration

The physical door lock connects to the software via the `/door/check` API. Each door unit runs an ESP32 microcontroller that calls the API on every PIN entry.

### Components (per door)

| Component                            | Estimated Cost (KES) |
| ------------------------------------ | -------------------- |
| ESP32 Dev Board (38-pin, Type C)     | 1,500                |
| 4x4 Matrix Keypad                    | 300                  |
| 0.96" OLED I2C Display (SSD1306)     | 400                  |
| 5V 1-Channel Relay Module            | 200                  |
| 12V Fail-Secure Electric Strike Lock | 3,000–5,000          |
| Active Buzzer Module                 | 150                  |
| 12V 2A Power Supply Adapter          | 800                  |
| Junction box + jumper wires          | 500                  |
| **Total**                            | **~7,000–9,000**     |

### Hardware Flow

```
Tenant presses PIN on keypad
        ↓
ESP32 collects digits
        ↓
ESP32 calls POST /door/check { unit_number, pin }
        ↓
API returns { "access": "granted" } or { "access": "denied" }
        ↓
Granted → relay triggers → electric strike releases → door opens 5s
Denied  → buzzer sounds → OLED displays ACCESS DENIED
```

### Installation Notes

- **Outside door:** Keypad + OLED display only (tamper-proof)
- **Inside (hidden):** ESP32 + relay + power supply
- **Electric strike** installs in door frame alongside existing manual lock
- **Manual key** always works as emergency backup
- **Exit** always works regardless of payment status
- **Fail-secure lock** stays locked on power failure — correct for rental security

## Payment Integration

**Webhook URL:**

**Citapay Webhook Payload:**

```json
{
  "event": "payment.completed",
  "data": {
    "reference": "TXNABC123",
    "amount": 3000,
    "status": "COMPLETED",
    "customerPhone": "254712345678",
    "metadata": {
      "unit_number": "A1",
      "tenant_id": "uuid-here"

  }
}
```

Tenant pays independently via M-Pesa → Lipa Na M-Pesa → Pay Bill → enters Paybill number + unit number as account reference → system updates access automatically. Requires a registered Safaricom Paybill shortcode.

## Security

- Passwords and PINs hashed with **bcrypt**
- Session tokens signed with **JWT** (httpOnly cookies, configurable expiry)
- Webhook requests verified with **HMAC-SHA256 signature**
- **Idempotency check** on all payment callbacks — duplicates logged and ignored
- **CORS** restricted to production domain only
- All secrets stored in **environment variables**
- `.env` excluded from version control via `.gitignore`
- Manual unlocks require **authorization confirmation** and are permanently logged with timestamp and user

## 🛠️ Troubleshooting

| Issue                                     | Possible Cause / Fix                                                                          |
| ----------------------------------------- | --------------------------------------------------------------------------------------------- |
| Webhook not firing after payment          | Check Citapay dashboard webhook URL and secret. Ensure server is reachable and not sleeping.  |
| Door not unlocking after payment          | Confirm payment processed, access expiry updated, and ESP32 is online. Check logs for errors. |
| STK Push not received on phone            | Verify phone number format (start with 254), check Citapay API status, and phone network.     |
| "Method Not Allowed" on /api/pay/confirm  | Ensure webhook is POST, not GET. Check Citapay webhook settings.                              |
| Render service sleeping / slow first load | Free tier spins down after 15 min idle. First request may take 30–60s. Upgrade for always-on. |
| Login not working after password reset    | Clear browser cookies, try incognito, or reset password again.                                |
| Tenant shows "Never paid" after payment   | Payment may not be linked to correct unit/tenant. Check payment reference and logs.           |
| ESP32 not connecting to API               | Check WiFi credentials, API URL, and SSL. Test API with Postman/cURL from same network.       |
